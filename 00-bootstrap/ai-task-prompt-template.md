# AI Task Prompt Template

這份模板用來啟動 Claude / Codex / 其他 AI coding agent。

目標是避免每次都重新說明規則，並確保 AI 工作流可控、可驗證、可交接。

## 通用任務模板

```txt
請先讀取專案結構與相關文件，再開始執行。

任務：
<在這裡描述要完成的功能、bugfix 或文件整理>

執行規則：
1. 不得自動 commit / push，除非我明確要求。
2. 修改前先確認目前專案結構、入口檔案、關聯模組。
3. 不要只改表面，要追資料流、參數流與呼叫鏈。
4. 修改後必須執行測試、build、lint 或至少做可說明的模擬驗證。
5. 如果無法測試，必須明確說明原因與替代驗證方式。
6. 回報時請列出：
   - 修改了哪些檔案
   - 解決了什麼問題
   - 如何驗證
   - 還有哪些風險或待確認事項
7. 遇到權限問題、環境問題、缺少資訊時，要先回報，不要假裝完成。
```

## Claude 主控 + Codex 審查模板

```txt
請讀取需求並開始執行。

你是主控 AI，負責實作與協調 Codex CLI 審查。

角色分工：
1. Claude：負責理解需求、修改程式、執行測試。
2. Codex：只負責審查 git diff，不准修改程式。

工作流程：
1. 先閱讀專案結構與關聯文件。
2. 完成第一輪修改。
3. 產生 git diff。
4. 呼叫 Codex CLI 審查 git diff。
5. 將 Codex 審查結果整理到 AI_REVIEW.md。
6. 依照審查結果修正。
7. 最多進行 3 輪。
8. 最後輸出交付報告。

限制：
- 不得自動 commit。
- 不得自動 push。
- 不得跳過測試。
- 不得讓 Codex 修改檔案。
- 如果 Codex 無法啟動，請回報原因與替代審查方式。
```

## 審查導向模板

```txt
請審查目前 git diff。

審查重點：
1. 是否有破壞既有功能。
2. 是否有安全性問題。
3. 是否有未處理的錯誤流程。
4. 是否有重複邏輯或過度設計。
5. 是否缺少測試或驗證。
6. 是否有命名、可讀性、維護性問題。

請輸出：
- Blocker
- Major
- Minor
- 建議修正方向
- 是否可以進入下一輪
```

## 回報格式

```md
## 完成摘要

## 修改檔案

## 驗證結果

## 風險與限制

## 下一步建議
```
